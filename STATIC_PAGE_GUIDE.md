# 정적 사이트에서 pid 파라미터로 동적 콘텐츠 표시하기

백엔드 데이터베이스 없이 정적 HTML에서 `pid` 파라미터로 콘텐츠를 동적으로 표시하는 방법입니다.

## 방법 비교

### 방법 1: JSON 파일 + fetch API (권장)
- **장점**: 콘텐츠 수정이 쉬움, 파일 분리로 관리 용이
- **단점**: JSON 파일을 별도로 로드해야 함
- **파일**: `m/page.html`, `m/data/pages.json`

### 방법 2: JavaScript 객체에 직접 저장
- **장점**: 외부 파일 로드 불필요, 빠름
- **단점**: HTML 파일이 커짐, 수정 시 HTML 파일 직접 수정 필요
- **파일**: `m/page_v2.html`

### 방법 3: jQuery 사용 (프로젝트에서 이미 사용 중)
- **장점**: 기존 프로젝트와 일관성, jQuery 기능 활용 가능
- **단점**: jQuery 라이브러리 필요
- **파일**: `m/page_jquery.html`

## 사용 예시

```
/m/page.html?pid=17  → 브랜드소개
/m/page.html?pid=44  → 가맹절차
/m/page.html?pid=45  → 가맹비용
/m/page.html?pid=56  → 사업비전
```

## 추천 방법

**프로젝트에서는 방법 2 또는 방법 3을 추천합니다:**
- 이미 jQuery를 사용하고 있음
- 정적 사이트이므로 외부 JSON 파일 로드가 복잡할 수 있음
- JavaScript 객체에 직접 저장하면 모든 것이 한 파일에 있어 관리가 쉬움

## 콘텐츠 추가 방법

`pagesData` 객체에 새로운 pid와 콘텐츠를 추가하면 됩니다:

```javascript
var pagesData = {
    "17": { "title": "브랜드소개", "content": "..." },
    "44": { "title": "가맹절차", "content": "..." },
    "45": { "title": "가맹비용", "content": "..." },
    "56": { "title": "사업비전", "content": "..." },
    "99": { "title": "새로운 페이지", "content": "<p>새 콘텐츠</p>" }  // 추가
};
```

## 주의사항

1. GitHub Pages 등 정적 호스팅에서도 동작합니다
2. 콘텐츠가 많아지면 HTML 파일이 커질 수 있으므로, 그 경우 JSON 파일 방식(방법 1)을 고려하세요
3. SEO를 위해서는 각 pid별로 별도 HTML 파일을 만드는 것도 고려할 수 있습니다
